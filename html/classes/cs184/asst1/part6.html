Part 6: Pixel sampling for texture mapping

In this part we make use of the barycentric coordinates to perform texture mapping. We use the texture triangle's coordinates in the uv space to create a bijective mapping for each barycentric coordinate point we attempt to rasterize by sampling from a texture for the appropriate color. There are two sampling techniques that we use: nearest and bilinear. Nearest sampling simply chooses the closest pixel in the texture to the uv coordinate we mapped to. Bilinear uses the four nearest pixels to perform two rounds of linear interpolation. The result is a bit of an averaging effect with the nearby pixels, with a focus on the closer pixels.

In this case, we have a well defined difference, because nearest sampling will simply choose the closest pixel. Since there is a bit of warping it appears that the sampler misses certain texels (such as the one creating the lines) and causes a bit of discontinuities in the image. Performing bilinear sampling serves to ameliorate these effects and makes the resulting image look much smoother. The biggest difference between these two functions will likely be in the case of magnification. This is because magnification will cause pixels to be dropped and cause the sharp discontinuities that we are witnessing now.

In this case, since we supersample, we end up with a similar image in both cases. Since we supersample with a sample rate of 16, the small amount of averaging from the interpolation on the texture performed by the bilinear sampling makes little difference in the face of the supersample averaging on these already averaged values.

In this final case there is very little difference between the two sampling techniques. This is likely because there is minification, so the image is already being downsampled, so introducing bilinear interpolation on the texture end makes little difference.
