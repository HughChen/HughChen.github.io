Part 7: Level sampling with mipmaps for texture mapping

For this part, we had to implement level sampling. This was basically a matter of passing in the barycentric coordinates of (x+1,y) and (x,y+1) into the color function, and then finding the uv coordinates of these guys in order to compute du/dx, dv/dx, du/dy, and dv/dy. Then we calculate the level according to the equation on the lecture slides. For the bilinear level case we simply use the levels adjacent to the float D we calculated to perform linear interpolation on the resulting samples. Using nearest and bilinear sampling and zero and nearest levels, I've shown a few examples for texmap using a png of a forest I found online. In this case there should be minification.

It's a bit hard to see, but for both sampling techniques, using the nearest level mipmap helps with the aliasing in the grass. This is because the grass originally had some random pieces of wood and other debris in it. When we minified, we created aliasing because we won't get all pixels of the debris in the rasterized image. Using the nearest level helps to blur this debris out.
